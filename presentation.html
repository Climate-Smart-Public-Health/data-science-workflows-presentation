Local & Manual Workflows are Fragile

Fragile Workflows ‚õìÔ∏è‚Äçüí•:

  • Stored only on personal machines
  • Manual file movements (click-and-drag)
  • Impossible to reproduce or share
  • Outputs treated as ‚Äútruth‚Äù

Robust Workflows üîó:

  • Portable across systems and environments
  • Can be automated
  • Replicable & reproducible
  • Source of truth is the ‚Äúhow‚Äù, not the ‚Äúwhat‚Äù
  • Robust workflows make doing the right thing the easy thing

Where is the source of truth? üßê

  • Claim: title
  • Evidence: data
  • Truth: methods

Proposing A Data Science Philosophy üí¨

  • Prefer remote + reproducible over local + manual
  • Code, not outputs, should be the source of truth
  • Use notebooks to narrate your thinking
  • Track your work with Git
  • Save on technical debt by investing in robust systems

Modularity in Data Science

  • üëØCollaboration: Modular code is easier for teams to understand, reuse, and extend.
  • How can you make your code more modular?
    • üß≠Portability of location: Using tools like here() to manage file and document paths
    • üèóÔ∏èPortability of dependencies: Using tools like renv or conda to manage package dependencies
    • üéÅPortability of code components: Using functions and modules so that code can be reused and tested independently
  • üìàScalability: Modular workflows adapt easily to new data, collaborators, or computing environments.

Version Control with Git

Why Git?

  • Track every change you make
  • Recover past versions
  • Collaborate without fear
git init
git add analysis.Rmd
git commit -m "Initial analysis"

Start with usethis::use_git() in RStudio

GitHub Integration

  • Create a private repo on GitHub
  • Link your local project:
git remote add origin https://github.com/yourname/project.git
git push -u origin main
  • Use issues and branches to track progress

What Belongs on GitHub?

‚úÖ Safe to Share

  • Code (.R, .Rmd, .qmd, .py)
  • Environment files (renv.lock, DESCRIPTION, .Rproj)
  • Documentation (README.md)
  • Simulated / anonymized results

‚ùå Do NOT Push

  • PII or protected health data
  • Raw survey or clinical data
  • Proprietary documents
  • .csv files with sensitive fields

Demo: Safe Project on GitHub

git commit -m "Initial reproducible analysis"

Where Should Outputs Go?

Type of File Go To
Analysis code, cleaned notebooks GitHub
Final tables for review (PII-safe) GitHub or Google Drive
Sensitive data outputs Google Drive via pins
Raw data from collaborators Google Drive (read-only)

results <- my_analysis_function(data)
